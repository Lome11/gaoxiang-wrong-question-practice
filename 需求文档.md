# 高项错题练习页面需求文档

## 1. 需求背景
为了帮助用户高效准备高项考试，需要一个专门的错题练习页面，让用户能够集中练习错题，提高学习效率。系统需要支持多种题型、本地存储错题、以及从外部导入题目等功能。

## 2. 功能清单
1. **题目展示**：支持选择题、判断题等多种题型展示
2. **答题功能**：用户可以选择答案并提交
3. **错题收集**：自动将答错的题目加入错题库
4. **本地存储**：使用本地存储保存题目和答题记录
5. **错题练习**：专门的错题练习模式，反复练习错题
6. **题目导入**：支持从豆包输出的格式直接导入题目
7. **界面交互**：干净易用的用户界面，良好的交互体验

## 3. 页面/模块结构

### 3.1 主页面整体布局
页面采用左右两栏布局：
- **左侧栏（答题卡区域）**：占较小宽度，展示答题进度和题目导航
- **右侧栏（题目区域）**：占主要宽度，展示当前题目详情、答题选项、解析等

### 3.2 左侧栏 - 答题卡模块

#### 3.2.1 计时器
- 位置：左侧栏顶部
- 显示内容：当前练习用时，格式为 `HH : MM : SS`
- 样式：数字使用橙色/黄色高亮显示，背景为浅色圆角卡片
- 功能：实时计时，记录用户本次练习的用时

#### 3.2.2 答题卡（题目导航）
- 标题："答题卡"
- 进度显示：显示当前题号/总题数，如 `15/235`
- 题目编号网格：
  - 以 5×7 网格形式展示所有题目编号（1-35+）
  - 每个编号为圆角矩形按钮
  - **状态颜色标识**：
    - 绿色：已答对
    - 红色：答错
    - 灰色/白色：未作答
  - 当前题目高亮显示（如第15题用红色背景突出）
  - 点击可快速跳转到对应题目
- 图例说明：底部显示颜色说明
  - 绿色圆点 + "答对"
  - 红色圆点 + "答错"
  - 空心方框 + "主观题"
- 提示文字："若有主观题，系统暂不支持自动判分"

#### 3.2.3 操作按钮
- **继续练习上次错题**：白色背景按钮，带边框
- **交卷**：绿色/青色背景按钮，醒目样式

### 3.3 右侧栏 - 题目详情模块

#### 3.3.1 页面标题
- 标题："错题本练习"
- 位置：页面顶部

#### 3.3.2 题目操作按钮组
- 位置：标题右侧
- 按钮列表：
  - **纠错**：图标 + 文字，用于反馈题目错误
  - **标记**：图标 + 文字，用于标记重要题目
  - **收藏**：图标 + 文字，用于收藏题目

#### 3.3.3 题目信息区
- 题号：如 `15`
- 题型标签：如 `单选题`，使用蓝色/青色标签样式

#### 3.3.4 题目内容区
- 题目文本：如 `（ ）不属于管理干系人参与的输入`
- 样式：正常字体，清晰可读

#### 3.3.5 选项区域
- 展示形式：单选按钮 + 选项文本
- 选项样式：
  - 正常状态：白色背景，带边框
  - 选中状态：浅绿色背景，带对勾图标
  - 错误状态：浅红色背景，带错误提示
- 选项内容示例：
  - A. 假设日志（正确答案，显示绿色背景和对勾）
  - B. 沟通管理计划
  - C. 变更日志（用户选择的错误答案，显示红色背景）
  - D. 问题日志

#### 3.3.6 答题结果反馈区
- 显示内容：
  - 红色标签："回答错误"
  - 正确答案："正确答案: A"
  - 用户答案："你的答案: C"
- 样式：紧凑排列，颜色区分

#### 3.3.7 解析区域
- 标题："解析"
- 内容：详细的知识点解析
- 示例内容：
  ```
  管理干系人参与的输入：
  1. 项目管理计划（沟通管理计划、风险管理计划、干系人参与计划、变更管理计划）
  2. 项目文件（问题日志、干系人登记册、变更日志、经验教训登记册）
  3. 事业环境因素
  4. 组织过程资产
  ```
- 样式：灰色背景区域，清晰排版

#### 3.3.8 广告/推荐区域（底部）
- 展示形式：卡片式推荐
- 内容：
  - 课程图片/头像
  - 课程标题："软考中高项备考特训营"
  - 讲师："薛大龙"
  - 时间："04/15 20:00"
  - 价格："¥0"
  - 按钮："了解课程"

### 3.4 错题管理模块
- 错题列表：展示所有错题
- 筛选功能：按题型、错误次数等筛选
- 清空功能：清空错题库

### 3.5 题目导入模块
- 导入界面：文本输入框，粘贴豆包输出的题目
- 解析按钮：解析并导入题目
- 导入结果：显示导入成功/失败信息

## 4. 交互流程

### 4.1 普通练习流程
1. 用户进入主页面，选择普通练习模式
2. 系统随机展示题目
3. 用户选择答案并提交
4. 系统判断答案是否正确
5. 若错误，将题目加入错题库
6. 显示正确答案和解析
7. 用户点击下一题，重复上述流程

### 4.2 错题练习流程
1. 用户进入主页面，选择错题练习模式
2. 系统从错题库中提取题目
3. 用户答题并提交
4. 若回答正确，可选择将该题从错题库中移除
5. 继续练习下一题

### 4.3 题目导入流程
1. 用户进入导入页面
2. 粘贴豆包输出的题目文本
3. 点击解析按钮
4. 系统解析题目并导入
5. 显示导入结果

### 4.4 答题卡交互流程
1. 用户点击左侧答题卡中的任意题号
2. 页面跳转到对应题目
3. 当前题目在答题卡中高亮显示
4. 答题后答题卡对应题号更新状态颜色

### 4.5 答题交互流程
1. 用户阅读题目内容
2. 点击选项进行选择（单选直接选中，多选可切换）
3. 系统自动判断答案（或点击提交按钮）
4. 显示答题结果：
   - 正确答案显示绿色背景 + 对勾
   - 错误选项显示红色背景
   - 显示"回答错误"标签
   - 显示正确答案和用户答案对比
5. 展开解析区域，展示详细解析
6. 用户可点击"下一题"或答题卡跳转到其他题目

### 4.6 计时器交互
1. 进入练习页面时计时器自动开始计时
2. 实时显示小时:分钟:秒
3. 交卷时停止计时并记录总用时

## 5. 业务规则

### 5.1 题目格式
- 支持选择题（单选、多选）和判断题
- 单选题：只有一个正确答案
- 多选题：有多个正确答案，需全部选对才算正确
- 判断题：正确/错误两个选项

### 5.2 答案判断
- 系统自动判断答案正确性
- 单选题/判断题：选择后立即判断或在提交时判断
- 多选题：需选择所有正确选项才算正确

### 5.3 错题收集
- 只有答错的题目才会被加入错题库
- 错题记录错误次数和最后错误时间
- 错题库中的题目可以重复练习

### 5.4 本地存储
- 使用localStorage存储题目和答题记录
- 存储键名：
  - `exam_questions`：所有题目
  - `exam_wrong_questions`：错题
  - `exam_answer_records`：答题记录

### 5.5 导入规则
- 支持豆包输出的标准格式
- 自动解析题目、选项、正确答案、解析

### 5.6 练习模式
- 普通练习模式：随机展示所有题目
- 错题练习模式：只展示错题库中的题目

### 5.7 答题卡状态规则
- **未作答**：灰色/白色背景
- **已答对**：绿色背景
- **已答错**：红色背景
- **当前题目**：高亮边框或特殊标识

### 5.8 计时规则
- 进入练习页面自动开始计时
- 格式：HH:MM:SS
- 交卷时停止计时并保存用时记录

### 5.9 解析展示规则
- 答题后自动展开解析区域
- 解析包含：正确答案说明、知识点详解、相关考点
- 解析区域可折叠/展开

### 5.10 收藏与标记规则
- **收藏**：用户可将重要题目加入收藏夹，方便后续复习
- **标记**：用于标记存疑或需要重点关注的题目
- **纠错**：用户可反馈题目错误，提交纠错信息

## 6. 数据存储说明

### 6.1 题目数据结构
```javascript
{
  id: String,              // 题目唯一标识
  type: String,            // 题型：single（单选）、multiple（多选）、judgment（判断）
  content: String,         // 题目内容
  options: Array,          // 选项列表，如 [{key: 'A', text: '假设日志'}, ...]
  correctAnswer: Array,    // 正确答案，如 ['A']
  explanation: String,     // 解析内容
  wrongCount: Number,      // 错误次数
  lastWrongTime: Date,     // 最后错误时间
  isCollected: Boolean,    // 是否收藏
  isMarked: Boolean,       // 是否标记
  category: String         // 题目分类/章节
}
```

### 6.2 答题记录数据结构
```javascript
{
  questionId: String,      // 题目ID
  userAnswer: Array,       // 用户答案
  isCorrect: Boolean,      // 是否正确
  answerTime: Date,        // 答题时间
  timeSpent: Number        // 答题用时（秒）
}
```

### 6.3 练习会话数据结构
```javascript
{
  sessionId: String,       // 会话ID
  mode: String,            // 练习模式：normal（普通）、wrong（错题）
  startTime: Date,         // 开始时间
  endTime: Date,           // 结束时间
  totalTime: Number,       // 总用时（秒）
  totalQuestions: Number,  // 总题数
  correctCount: Number,    // 正确数
  wrongCount: Number       // 错误数
}
```

### 6.4 存储方式
使用localStorage存储，键名分别为：
- `exam_questions`：所有题目
- `exam_wrong_questions`：错题
- `exam_answer_records`：答题记录
- `exam_sessions`：练习会话记录
- `exam_settings`：用户设置（如计时器开关、显示解析方式等）

## 7. 前端需要做什么

### 7.1 页面布局
- 设计干净美观的界面，响应式布局
- 左右两栏布局：左侧答题卡（固定宽度），右侧题目区（自适应）
- 支持不同屏幕尺寸的适配

### 7.2 组件拆分
建议拆分为以下组件：
1. **Timer组件**：计时器显示
2. **AnswerCard组件**：答题卡网格
3. **QuestionCard组件**：题目展示卡片
4. **OptionList组件**：选项列表
5. **ResultFeedback组件**：答题结果反馈
6. **ExplanationPanel组件**：解析面板
7. **ActionButtons组件**：操作按钮组（纠错、标记、收藏）

### 7.3 交互实现
- 实现答题、提交、切换题目等交互功能
- 答题卡点击跳转
- 选项选择交互
- 计时器自动计时
- 交卷功能

### 7.4 本地存储
- 使用localStorage进行数据存储和读取
- 封装存储工具函数
- 处理存储容量限制

### 7.5 题目解析
- 实现从豆包格式解析题目的功能
- 支持批量导入
- 解析错误处理

### 7.6 状态管理
- 管理练习状态、错题状态等
- 使用Vuex/Pinia或React Context进行状态管理

### 7.7 动画效果
- 选项选择动画
- 答题结果反馈动画
- 页面切换过渡动画
- 答题卡状态变化动画

### 7.8 UI样式要求
根据参考图片，样式要求如下：
1. **整体风格**：简洁、清新、专业
2. **配色方案**：
   - 主色调：绿色（#52c41a 或类似）表示正确
   - 错误色：红色（#ff4d4f 或类似）表示错误
   - 背景色：浅灰色（#f5f5f5）
   - 卡片背景：白色
   - 强调色：橙色/黄色用于计时器
3. **字体**：清晰易读，标题加粗
4. **间距**：适当的留白，不拥挤
5. **圆角**：按钮和卡片使用圆角设计
6. **阴影**：卡片使用轻微阴影增加层次感

## 8. 后端需要做什么
由于是纯前端应用，不需要后端服务。所有数据都存储在本地。

## 9. 测试点
1. **功能测试**
   - 题目展示是否正确
   - 答题功能是否正常
   - 错题是否正确加入错题库
   - 本地存储是否正常工作
   - 错题练习模式是否只展示错题
   - 题目导入功能是否正常

2. **边界测试**
   - 空题目列表的处理
   - 导入格式错误的处理
   - 本地存储容量限制的处理
   - 重复导入题目的处理

3. **性能测试**
   - 大量题目时的加载速度
   - 本地存储读写性能

## 10. 风险与注意事项
1. **本地存储限制**：localStorage有容量限制（约5MB），需要注意题目数量
2. **数据安全**：本地存储数据可能被清除，建议定期备份
3. **导入格式**：需要确保豆包输出的格式符合预期，否则可能解析失败
4. **兼容性**：需要考虑不同浏览器的兼容性
5. **用户体验**：需要确保界面简洁易用，避免复杂操作
